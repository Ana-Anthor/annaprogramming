How to create a SSH key to use in GitHub

Key exchange:

If you want to copy a repository with ssh and not https, you first have to exchange ssh keys. I have written two guides, one for windows and one for iOS/Linux. If you have already made an ssh key, but you have forgotten your password, you can just make a new one following this guide and it will overwrite your old one.

  • For Windows
  • For iOS/Linux

For Windows:

Press Windows key Open command prompt by typing "cmd", right click on "Command Prompt" and choose "run as administrator". Navigate to your home location, or the place you want to store your ssh key. 

$ cd %HOMEPATH%

or

$ cd C:\Users\yourUser

To create a ssh key, write:

$ ssh-keygen

You will now get this question:

$ Enter file in which to save the key (C:\Users\yourUser/.ssh/id_25519):

Press enter to save the key in the default directory C:\Users\yourUser/.ssh (If you have an old key you will get a message telling you it already exists and asks you if you want to overwrite is. Write y for yes.) Enter passphrase and choose something you can remember, or just press enter if you don't want to use a passphrase. Press enter until you see the key's randomart image. Navigate into the the .ssh folder 

$ cd .ssh

Copy your public key:

$ type id_25519 .pub | clip

If you are using git bash you can copy using cat:

$ cat id_ed25519.pub | clip

Then go to GitHub. Click on your profile image in the upper right corner and select "Settings". In the left margin choose "SSH and GPG keys". Click on "New SSH key".

  • Title: Choose a title, for instance "My PC"
  • Key type: Anthentication key
  • Key: Paste the key you copied in the key field and press the button "Add SSH key"
